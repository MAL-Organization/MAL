◆ MAL_TIMER_SUB_COUNTS

#define MAL_TIMER_SUB_COUNTS (   count2,
  count1,
  mask 
)    (((count2) + ((-count1) & mask)) & mask)

Macro to help subtract 2 timer values regardless of the resolution. The result will be count2 - count1 respecting the two's complement wrap around. This means that for an 8 bit resolution timer, 1 - 255 = 2.

Parameters
count2The count 2 value. Should be the most recent count.
count1The count 1 value. Should be the oldest count.
maskThe value obtained by mal_timer_get_count_mask.
Returns
The result of the subtraction.

◆ mal_timer_direct_init_tick()

mal_error_e mal_timer_direct_init_tick ( mal_hspec_timer_e  timer,
mal_hertz_t  frequency,
mal_hertz_t  delta,
const void *  direct_init,
mal_hspec_timer_e *  handle 
)

Function to initialize directly the timer. Using this function will reduce code size at the cost of flexibility and safety. Usually use this in a code closer to final.

Parameters
timerThe desired timer to initialize. It is not recommended to use timer ANY when using this function.
frequencyThe frequency to count at.
deltaThe allowed delta of frequency. This means the actual frequency of the timer can be frequency +/- delta.
direct_initA pointer to direct initialization parameter. See the hardware specific implementation to know what type this should be.
handleThis handle will return the used timer. Useful when using MAL_HSPEC_TIMER_ANY.
Returns
MAL_ERROR_OK on success.

◆ mal_timer_init_count()

mal_error_e mal_timer_init_count ( mal_hspec_timer_e  timer,
mal_hertz_t  frequency,
mal_hspec_timer_e *  handle 
)

This is a simple timer initialization. The main difference between this and the tick timer is you have to handle yourself the overflow of the timer based on timer resolution. This method is much faster however because it doesn't rely on interrupts to work. Use this to time precise events.

Parameters
timerThe desired timer to initialize.
frequencyThe frequency to count at.
handleThis handle will return the used timer. Useful when using MAL_HSPEC_TIMER_ANY.
Returns
MAL_ERROR_OK on success.
